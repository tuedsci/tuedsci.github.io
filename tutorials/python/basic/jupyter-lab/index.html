Introduction to Jupyter Lab

What is Jupyter Lab?

Jupyter Lab is an integrated development environment (IDE) commonly used for interactive coding with languages such as Python, R, and Julia. In fact, Jupyter is an acronym for Julia-Python-R, the three most popular languages used in data science. You can also you Jupyter Lab with other scripting languages such as MATLAB or Octave.

An IDE is simply a text editor with some features that make coding easier and more efficiently such as

  • Syntax highlighting
  • File organization/navigation
  • Code execution
  • Integrated terminal
  • Debugger

Jupyter Lab is the next generation of Jupyter Notebook, and it fully supports Jupyter Notebook. However, it has a better GUI and many more features and enhancements that boost your productivity.

Besides Jupyter Lab, there are many excellent alternatives such as PyCharm, Spyder, Visual Studio Code, or Zeppelin. You are free to explore them if you want, but my advice is to stick with Jupyter Lab in your early stages of learning Python.

Installation

If you followed my instructions and installed Anaconda as presented in the previous lesson, then you don’t need to do anything because Jupyter Lab was already installed alongside Anaconda.

Start Jupyter Lab

The best approach is to start Jupyter Lab from the root directory of your project. For Windows users, follow the instructions below.

Step 1: go inside your project, for example, pycourse/.

Step 2: right-click on a blank space and choose Git Bash Here from the context menu. On Windows 11, you have to choose Show more options first, then you will see Git Bash Here.

Step 3: a Git Bash terminal will show up, type jupyter lab and hit Enter. After a few seconds, a new tab will open on your default browser; that’s the interface of Jupyter Lab.

Note: I recommend using a modern browser such as Edge, Chrome, or Firefox for a better experience.

Create your first analysis

Your project can contain only a single notebook or a collection of notebooks, depending on the complexity of your analysis. Each notebook is a piece of analysis that contains code and explanatory text. It has the extension .ipynb, which stands for IPython Notebook (the very first version of Jupyter Lab).

To create a notebook, follow the instructions below.

Step 1: on the navigation panel on the left, click on the big plus button. You will see a Launcher tab show up.

Step 2: on the Launcher, click on the icon with the caption Python 3 (ipykernel). You will see a new notebook named Untitled.ipynb created.

Step 3: right-click on the file name, and choose Rename to change the file name.

Step 4: write some simple expressions into the first cell in the notebook, for example, 1 + 1, and hit Ctrl + Enter to execute it. You will see the result 2 printed out.

Markdown and code cells

There are two types of cells in a notebook: code cells and Markdown cells.

Code cells

Code cells are used to write Python code. Therefore, what you write must conform to Python syntax. Otherwise, you will get an error when executing the cell.

Markdown cells

Markdown cells are used to write explanatory texts. You will not get any error when executing a Markdown cell. However, for your texts to be rendered as intended, you should follow the Markdown syntax, which we will discuss in the next lesson.

Editing and command modes

When working with notebook cells, you are either in the editing mode or in the command mode.

Editing mode

In this mode, you just write something inside a cell such as some Python code or explanatory texts. To activate this mode, just double-click into a cell and start typing.

Command mode

In this mode, you perform some action on an entire cell or a set of cells such as copying, cutting, or deleting. To activate this mode, you put the cursor at a cell and hit Esc followed by another key dedicated to a certain action. For example, Esc + C is for copying, Esc + V is for pasting, and so on. More on this in the next section.

Jupyter shortcuts

You can interact with Jupyter Lab using buttons, but that way is not efficient. Use your keyboard as much as you can because it will help you code faster (and with more fun).

The following are some of the shortcuts that I frequently use in my daily work. I will divide them into groups so that it will be easier for you to remember.

Basic cell manipulation

  • Esc + A: insert a cell ABOVE the current cell
  • Esc + B: insert a cell BELOW the current cell
  • Esc + DD: DELETE the current cell (remember to hit D twice with lightning speed)
  • Esc + C: COPY the current cell
  • Esc + X: CUT the current cell
  • Esc + V: PASTE the copied/cut cell BELOW the current cell
  • Esc + Z: UNDO the last action
  • Esc + Y: convert the current cell to a CODE cell
  • Esc + M: convert the current cell to a MARKDOWN cell

Note: you will learn about Markdown in the next lesson.

Cell execution

  • Ctrl + Enter: run the current cell and stay at the cell
  • Shift + Enter: run the current cell and advance to the next one
  • Alt + Enter: run a cell, insert a new one below it, and advance to this new cell

Final notes

Therefore, this lesson just gives you an idea of an appropriate tool for you to start with, not detailed instructions on how to use the tool.
